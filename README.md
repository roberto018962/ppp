# Princípios e Padrões de Projeto 🏗️

Repositório dedicado aos estudos de arquitetura, princípios de design e padrões de projeto (Design Patterns) realizados no 4º semestre do curso no **IFRS**.

## 🎯 Objetivo
Desenvolver a habilidade de projetar sistemas robustos, escaláveis e de fácil manutenção, aplicando conceitos avançados de Orientação a Objetos e padrões consolidados na indústria.

## 🧠 Conteúdos Principais

### 💎 Princípios de Design (SOLID)
Estudos e exemplos práticos dos princípios:
* **S** - Single Responsibility (Responsabilidade Única)
* **O** - Open/Closed (Aberto/Fechado)
* **L** - Liskov Substitution (Substituição de Liskov)
* **I** - Interface Segregation (Segregação de Interface)
* **D** - Dependency Inversion (Inversão de Dependência)

### 🧩 Padrões de Projeto (GoF - Gang of Four)
Implementações dos principais padrões categorizados por:
* **Criacionais:** Factory Method, Abstract Factory, Singleton, etc.
* **Estruturais:** Adapter, Composite, Proxy, Decorator, etc.
* **Comportamentais:** Strategy, Observer, **Template Method**, State, etc.

### 🧹 Refatoração e Código Limpo
* Técnicas para transformar "código sujo" em código legível e eficiente.
* Identificação de **Code Smells** e aplicação de padrões de refatoração.
* Noções de Anti-padrões e como evitá-los.

## 📂 Organização do Repositório
* `/principios`: Exemplos de violação e aplicação de SOLID.
* `/padroes`: Implementações separadas por tipo (Criacionais, Estruturais, Comportamentais).
* `/refatoracao`: Projetos de "antes e depois" de processos de refatoração.
* `/exercicios`: Práticas de fixação propostas em aula.

## 🛠️ Tecnologias e Referências
* **Linguagens:** Java
* **UML:** Diagramas de classe para representação dos padrões.
* **Livros Base:** * *Design Patterns* (Erich Gamma et al.)
    * *Código Limpo* (Robert C. Martin)
    * *Refatoração* (Martin Fowler)
